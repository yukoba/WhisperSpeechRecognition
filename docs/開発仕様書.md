# 開発仕様書：音声認識＆クリップボード自動入力 常駐アプリ

## 1. プロジェクト概要
Windows 11環境で動作する、音声認識を活用した常駐型アプリケーションを開発する。
グローバルホットキーで録音の開始・終了を切り替え、Gemini (gemini-3-flash-preview) API で文字起こしと「えっと」などのフィラー（ケバ）除去を一括で実行し、結果をクリップボードに自動設定する。

## 2. 技術スタック
- **ターゲットフレームワーク**: .NET 10 / WPF (Windows Presentation Foundation)
- **言語**: C#
- **主要ライブラリ (NuGet)**:
  - `NAudio`: マイク音声のキャプチャおよびWAVファイル生成
  - `NHotkey.Wpf`: グローバルホットキーの登録と検知
  - `Google.GenAI`: Gemini API通信
  - `Hardcodet.NotifyIcon.Wpf` (または標準の `NotifyIcon`): タスクトレイ（通知領域）への常駐
  - `Microsoft.Extensions.Configuration`: ユーザー設定（APIキー等）の保存と読み込み

## 3. 画面・UI仕様
本アプリはメインウィンドウを持たず、タスクトレイ常駐およびオーバーレイUIのみで動作する。

- **タスクトレイアイコン**:
  - アプリ起動時はタスクトレイにアイコンを表示し、バックグラウンドで待機する。
  - 右クリックメニュー（コンテキストメニュー）:
    - 「設定」: Gemini APIキーなどを設定する小さなウィンドウを表示。
    - 「終了」: アプリを完全に終了する。
- **録音中オーバーレイ表示**:
  - 録音中であることを示すインジケーター（「🎤 録音中...」というテキスト）。
  - 仕様: 常に最前面（`Topmost="True"`）、背景透明（`AllowsTransparency="True"`）、枠なし（`WindowStyle="None"`）。画面の中央下部に配置し、マウスクリックを透過する（`IsHitTestVisible="False"`）。
  - 録音開始ホットキーで表示（フェードイン）し、録音終了ホットキーで非表示（フェードアウト）にする。

## 4. 機能要件・動作フロー
1. **待機状態**:
   - アプリ起動後、グローバルホットキー（例：`Ctrl + Shift + Space`。設定で変更可能が望ましいが、初期は固定でも可）をシステムに登録し、待機する。
2. **録音開始**:
   - ユーザーがホットキーを押下する。
   - 録音中オーバーレイUIを表示する。
   - `NAudio` を使用して、既定のマイクデバイスから音声のキャプチャを開始する（メモリまたは一時ファイルに保存）。
3. **録音終了と処理開始**:
   - ユーザーが再度同じホットキーを押下する。
   - 録音中オーバーレイUIの表示を「⏳ 処理中...」に変更、または非表示にする。
   - 音声のキャプチャを停止し、WAVデータ（16kHz, 16bit, モノラル）にする。
4. **音声認識とフィラー除去・テキスト整形 (Gemini gemini-3-flash-preview)**:
   - WAVデータとプロンプトを同時に Gemini API (`gemini-3-flash-preview`) に送信する。
   - プロンプト: 「音声の内容を文字起こししてください。その際、『えっと』や『あの』などのフィラーはすべて除去し、読みやすい文章に整えてください。整形後のテキストのみを出力し、解説や挨拶は含めないでください。」
   - その際、Thinking levelは minimal に設定し、文字起こしとフィラー除去を一括で行う。
5. **クリップボードへの設定**:
   - 取得した最終的なテキストを Windows のクリップボードに設定する。
   - 処理完了をシステム音（`SystemSounds.Asterisk.Play()`など）で通知する。

## 5. 推奨クラス構成（アーキテクチャ）
AIエージェントは以下の責務分割を意識して実装すること。

- `App.xaml / App.xaml.cs`: アプリケーションのエントリーポイント。複数起動の防止（Mutex）、タスクトレイアイコンの初期化、DIコンテナのセットアップ（必要であれば）。
- `Services/AudioRecorder.cs`: `NAudio` をラップし、録音の開始・停止・WAVデータ取得を担うクラス。
- `Services/GeminiService.cs`: Gemini API との通信を担うクラス。
- `Services/HotkeyManager.cs`: `NHotkey` を用いたホットキーの登録とイベント発火を管理するクラス。
- `ViewModels/TrayIconViewModel.cs`: タスクトレイのメニュー操作や状態管理を担う。
- `Views/OverlayWindow.xaml`: 録音状態を表示する透明なビュー。

## 6. AIエージェントへの実装上の注意点（制約事項）
- **クリップボードのSTAスレッド制約**:
  - 非同期処理（`Task`）の完了後に `Clipboard.SetText()` を呼び出す際、必ず UI スレッド（STAスレッド）にディスパッチして実行すること。（例: `Application.Current.Dispatcher.Invoke(() => Clipboard.SetText(text));`）
- **例外ハンドリング**:
  - ネットワークエラー、APIキーの未設定、マイクのアクセス権限エラー、録音デバイスが存在しないエラーを適切に `try-catch` で捕捉し、ユーザーにトースト通知（またはタスクトレイのバルーン通知）で知らせること。アプリがクラッシュしないようにする。
- **メモリ管理**:
  - `NAudio` の `WaveInEvent` や一時ファイルは、録音終了時に適切に `Dispose` してリソースリークを防ぐこと。
- **環境変数・設定**:
  - Gemini APIキーは `%USERPROFILE%\AudioTranscription.json` に保存・読み込みできるようにすること。ソースコードへのハードコードは避ける。

## 7. 開発ステップ (AIへの指示用)
以下のステップ順に実装を進めてください。一度に全て書かず、ステップごとに動作確認可能な状態にしてください。

- **Step 1**: WPFプロジェクトの作成、タスクトレイへの常駐化（メインウィンドウを表示しない設定）。
- **Step 2**: グローバルホットキーの登録と、録音中オーバーレイUIの表示・非表示のトグル実装。
- **Step 3**: `NAudio` を用いたマイク録音の開始、停止、WAVファイル（一時ファイル）の保存機能の実装。
- **Step 4**: Gemini APIへの通信処理の実装。
- **Step 5**: 一連のフローを結合（録音 -> Geminiで一括処理 -> クリップボード設定）し、スレッドセーフなクリップボード操作を実装。
